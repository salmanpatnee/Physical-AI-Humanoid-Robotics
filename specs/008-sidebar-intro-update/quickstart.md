# Quickstart Guide: Docusaurus Sidebar and Introduction Update

This guide provides instructions for updating the Docusaurus sidebar and introductory content to reflect the custom book structure.

## 1. Introductory Content (`docs/intro.md`)

The primary introductory content for the book will reside in a standalone MDX file:

-   **File Path**: `docs/intro.md`

### Content Requirements:
The `intro.md` file MUST contain:
-   A high-level course outline.
-   Author introductions.
-   Prerequisites for the entire course.
-   Accurate and relevant information for the "Teaching Physical AI & Humanoid Robotics Course".

### Frontmatter Example for `docs/intro.md`:
```markdown
---
id: intro
title: Introduction
sidebar_position: 1
slug: /
description: An introduction to the Teaching Physical AI & Humanoid Robotics Course.
---
```
*Note*: Setting `slug: /` makes this the default content for the homepage. `sidebar_position: 1` places it at the top of the sidebar.

## 2. Docusaurus Sidebar Configuration (`sidebars.ts`)

The `sidebars.ts` file is central to defining the book's navigation structure. We will utilize Docusaurus's built-in "autogenerated sidebar" feature.

### Steps to Update `sidebars.ts`:

1.  Open `sidebars.ts` in the project root.
2.  Remove all default/placeholder sidebar items (e.g., "tutorialSidebar").
3.  Define a new sidebar array that includes `docs/intro.md` as the first item, followed by categories for each module.

### Example `sidebars.ts` Structure:
```typescript
import type { SidebarsConfig } from '@docusaurus/plugin-content-docs';

const sidebars: SidebarsConfig = {
  // By default, Docusaurus generates a sidebar from the docs folder structure
  tutorialSidebar: [
    {
      type: 'doc',
      id: 'intro', // This links to docs/intro.md
      label: 'Introduction',
    },
    {
      type: 'autogenerated',
      dirName: 'module1-robotic-nervous-system', // Generate sidebar from docs/module1-robotic-nervous-system
    },
    {
      type: 'autogenerated',
      dirName: 'module2-the-digital-twin', // Generate sidebar from docs/module2-the-digital-twin
    },
    {
      type: 'autogenerated',
      dirName: '006-module3-ai-robot-brain', // Generate sidebar from docs/006-module3-ai-robot-brain
    },
    {
      type: 'autogenerated',
      dirName: '007-module4-vla', // Generate sidebar from docs/007-module4-vla
    },
    // Add more modules here as they are created
  ],
};

export default sidebars;
```
*Note*: The `dirName` properties automatically pull chapters from the specified directories, using `_category_.json` for category labels and ordering.

## 3. Module Category Configuration (`_category_.json`)

Each module directory (e.g., `docs/module1-robotic-nervous-system/`) MUST contain an `_category_.json` file to define its sidebar heading and position.

### Example `_category_.json` for a Module:
```json
{
  "label": "Module 1: The Robotic Nervous System (ROS 2)",
  "position": 1,
  "link": {
    "type": "generated-index",
    "title": "Module 1 Overview"
  }
}
```
*Note*: Ensure `position` reflects the desired order of modules in the sidebar. The `link` property can create an auto-generated index page for the module if desired.

## 4. Docusaurus Build & Validation

To ensure your changes are correctly formatted and valid, run the Docusaurus build process locally:

```bash
npm run build
```
This command will validate the sidebar configuration, frontmatter, and check for any structural errors. Address any warnings or errors reported during the build.

## 5. Local Development Server

To preview your changes in real-time:

```bash
npm run start
```
Then navigate to `http://localhost:3000/`. Visually inspect the sidebar and introductory content.
