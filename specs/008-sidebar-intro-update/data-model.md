# Data Model for Docusaurus Sidebar and Introduction Update

This document outlines the key entities and their configurations related to the Docusaurus sidebar and introductory content within the project.

## Entities

### 1. Sidebar
Represents the primary navigation component in Docusaurus.

-   **Configuration File**: `sidebars.ts`
-   **Structure**: Defined as an array of sidebar items, where each item can be a category (module heading) or a link to a document (chapter).
-   **Key Property**: `items` (array of sidebar items).
-   **Generation**: Will primarily utilize Docusaurus's built-in "autogenerated sidebar" feature for hierarchical grouping based on file system and `_category_.json` files.
-   **Content**: Only book-specific content; all default/placeholder items will be removed.

### 2. Module Heading (Category)
Represents a top-level grouping for chapters within the sidebar.

-   **Configuration File**: `docs/NN-module-name/_category_.json`
-   **Properties**:
    -   `label`: Display name for the module (e.g., "Module 1: The Robotic Nervous System (ROS 2)").
    -   `position`: Numeric order of the module within the sidebar.
    -   `link`: (Optional) Can link to an overview page for the module.
    -   `collapsed`: (Boolean) Whether the category is collapsed by default.
-   **Relationship**: Contains an ordered list of Chapter entities (documents) under it, automatically generated from the folder structure.

### 3. Chapter (Document)
Represents an individual learning document within a Module, typically a single MDX file.

-   **File**: `docs/NN-module-name/NN-chapter-title.mdx`
-   **Frontmatter Metadata**:
    -   `id`: Unique identifier (e.g., `focus-middleware-for-robot-control`).
    -   `title`: Display title of the chapter.
    -   `sidebar_position`: Numeric order within its module category.
    -   `slug`: URL-friendly path.
    -   `description`: Short chapter summary.
    -   `category`: Reference to the parent module's category.
    -   Other custom metadata.
-   **Relationship**: Belongs to a Module Heading.

### 4. Introductory Content
The primary welcome and overview section of the book.

-   **File**: `docs/intro.md` (standalone MDX page)
-   **Frontmatter Metadata**:
    -   `id`: `intro`
    -   `title`: "Introduction"
    -   `sidebar_position`: `1` (or appropriate to be the first item).
    -   `slug`: `/` (can be configured to be the root URL).
    -   `description`: Overview of the course book.
-   **Content**: Must include a high-level course outline, author introductions, and prerequisites for the entire course. Can be rendered on the homepage.
-   **Relationship**: The first top-level item in the sidebar, preceding all Module Headings.
